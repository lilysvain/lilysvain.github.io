个人主页---全站

开始于：2012-06-07，结束于：2015-03-08

作品链接

花了一个寒假加开学一周，终于把新网站搞完了。原来网站几乎没写什么后台，更文非常不方便，自己又懒癌晚期碰上最为忙碌的大三，改版计划一直在搁浅。现在也是因为找工作中，就抽了抽懒筋，设计还是懒得做，以前的图能用就用了，一边 coding 一边 designing 。我不会告诉你我配色时都是用的 Firefox 上的 Rainbow 插件看到浏览器上顺眼的颜色就取。好啦~废话说的有点多了，下面来对旧网站做个总结。

Section1. 关于个人主页怎么搭

原来也是有些人知道我有这个主页，然后就常有人来问我，网站要怎么搭。简单的说呢，首先，你要有个服务器，然后要弄个域名，接着……接着我也不好说了。

其实如果不是当初入了 Jsp 的大坑，又舍不得弃了，我可能会去 SEA 上申请空间，或者找找看有没有支持 NodeJs 的。因为 Java Web 的学习周期实在太长了，而且我还有点想学下 python 和 NodeJs 写后台。但是强迫症使然，既然有了开始就不想草草结束，于是在 Java Web 之路上一去不复返了。反正我现在的技术中心都要前端上，后台写的好不好目前我不太关心，能用最快的速度搭出一个后台且方便我维护就行了。但是 Jsp 空间真的好贵……

至于申请域名呢，真的很简单。我服务器方面的都不怎么懂，这些事情也都是一个个百度出来的，找了个 Jsp 空间提供商，还有域名备案也是我提交资料给他们他们去办理的。还有域名解析什么的，照着找到的步骤一步步来就好了。

最重要的部分还是网站的内容啦。其实“设计+前端+后台”都自己来做非常非常累，我也是上大学后阴差阳错地这三个部分都有涉足，才能较快的整出个网站来。只学后台的人可以用 bootstrap，只会前端的人可以做静态页面啦，只会设计的人可以找人帮忙写个静态网站，至于都不怎么会又没有时间学的建议用用 wordpress 这样的博客平台。

Section2. 页面设计

当初的设计目标是很理想的，希望不用网上下载的图片素材，图标和纹理都自己 Ps，做一个精致的网站。但当时做着做着就失去了耐心。那个时候正是我从设计转为写代码的时期，因为我感觉设计做的很累，一直都是帮别人 P 图，我的手绘和配色来不及练习，身在工科专业也很难有那个氛围和时间。

从首页的图里可以看出来咯，上半部的小三角箭头还能细心的做阴影，下半部就全是色块了。至于其余的页面图，只剩链接页面也完全是自己的设计，文章、留言等等的都是以前自学网页设计时跟着外文教程 Ps 出来的图。

国外的这种设计做的非常精细，配色也看起来很舒服，但一个页面不思考地做下来都要花很多时间。

Section2. 前端开发

在写旧网站的时候，我前端还没接触多少，却尝试着用上了 html5 新的语义化标签，现在看来有些地方是使用不当的。不过我至今对 header、article 和 section 的使用还是有些疑惑，更完这篇文后，我打算对 html 标签写个总结。

Js 当时用的 JQuery，因为那会只会 JQuery，首页顶部的幻灯动画都是现学现用的 JQuery。当然新网站打算全部使用原生 Js 啦，因为我想建自己的库出来，不过兼容性的问题真的挺棘手。

旧网站的动画效果几乎为0，除了首页也就 Link 页面有个下拉子菜单的效果了，还全是用 CSS 实现的。

新网站的开发重心就会放在前端上了，CSS 使用了 compass + sass，Js 也想试试模块化开发的，但是我现在的电脑系统不太好用，NodeJs 死活安装不上，实在不行我就要重装系统了。

Section2. 前端开发

后台简陋，数据都放在数据库中，数据库是很简单的一维结构。后台仅仅是将数据库中的内容丢出来显示，所以评论和分类查找的功能当时都做不出来，发表和编辑文章也特别麻烦。虽然现在的网站仍很简陋，但用起来已经方便太多了。

网站的后台我写的挺纠结的。一度想放弃了 Jsp，或者使用 Struts2 + Spring 快速开发。事实证明 Java Web 的框架不是那么好上手的，而且国内这方面的书籍资料特别少，即使有也跟不上更新了，再者我嫌框架太笨重，最终还是用的最简朴的 Jsp + Servlet + JavaBean。
