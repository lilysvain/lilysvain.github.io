Android 学习笔记

该文会记录在学习和开发 Android 时遇到的一些问题以及解决方案。欢迎提出你所遇到的问题（可在文章后留言），如果找到合适的解决方法或者可参考的资料，我会在文章后面进行更新。

1. Android 开发环境配置

2. 安装Genymotion插件

因为 IDEA 自带的 Emulator 莫名用不了，而且也不能模拟蓝牙的功能，就安装了个 Genymotion 用来试试。

首先，要在自己的电脑上安装好 Genymotion，然后在 IDEA 中选择 File --> Settings，选择 Plugins 并点击 Browse Repositories。接着在左侧列表里找到 Genymotion 并点击安装。安装好后 IDEA 重启，View --> Toolbar 可以打开插件工具栏。

3. SDK 版本选择及安装选项

运行你安装好的 SDK Manager，可以从中选择你要安装的包。

Android SDK Tools —— 这个运行你的SDK所必须的包（必装）

Android SDK Platform-tools —— 在第一次运行 SDK Manager 时就安装好了

SDK Platform —— 至少有一个平台被安装。例如选择 Android 6.0（API23）下的 SDK Platform（必装）

*** System Image —— 不同体系结构的系统映像，这样你就可以在 Android 模拟器上测试应用程序（选装）

Google 官方强烈建议只用最新的 SDK 版本，新的 SDK 会提供更强大开发工具，编译的效率也更高。Android N 是2016年3月第一 次发布的新版本，它的稳定版年中才会推出，因此我推荐你们安装 Android 6.0（API23）

你的 APP 能运行的 Android 版本是由每一个项目的 minSDK 决定的，minSDK 的选择应该根据应用具体的 API 来，如果 app 没有 使用新版本 SDK 提供的 API，那么选择旧版本反而能兼容更多机型。4.2（API 17）、4.1（API16） 及更早的都可以，只是在开发的 过程中可能需要使用 support library，因此Extras 下的 “Android Support Library” 和 “Local Maven repository for Support Library” 也要安装上。

SDK Manager 的包管理是非常灵活的，可以随时对包进行安装和删除，所以不用担心装多或者装错。（图中我也有一些安装多余的）
